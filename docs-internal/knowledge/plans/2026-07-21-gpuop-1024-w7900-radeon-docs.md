# GPUOP-1024: metricslist.md W7900/Radeon platform support corrections

- **Date:** 2026-07-21
- **Author:** Bhanu Kiran Atturu
- **Related PR(s):** docs/gpuop-1024-w7900-radeon-support
- **Related issue(s) / JIRA:** GPUOP-1024

## Context

DME v1.5.1-2 on a W7900 (Radeon Pro, gfx1100/RDNA3) OpenShift cluster exports 6
metrics whose platform support in `metricslist.md` is stale, causing
`test_metric_coverage` to report untracked metrics. Evidence: CI job 32412610,
4 GPUs, driver 31.40, all 6 metrics present across 40 curl samples.

Of the 6 metrics flagged, only 2 are genuine doc errors (contradictions between
doc and observed emission). The other 4 already read correctly in the prose.

## Approach

Two edits to `docs/configuration/metricslist.md`:

- `GPU_AVERAGE_PACKAGE_POWER` (was `[MI2xx]`): add `Radeon`. W7900 exports valid
  non-zero values (7-9W idle) — not MI2xx-only.
- `GPU_PROF_WRITE_SIZE` (was "not exported" on Radeon): correct to "exported but
  always returns 0", matching the sibling `GPU_PROF_FETCH_SIZE` wording and the
  observed W7900 behaviour (exported, value 0).

One edit to `docs/configuration/metrics-support-matrix.yaml`:

- `GPU_AVERAGE_PACKAGE_POWER`: add `Radeon` to platforms (it exports real
  non-zero values there, so it's normal support — class stays `C`). The profiler
  metrics are out of this matrix's scope (`GPU_PROF_*` excluded by design, per
  file header).

No change for the 4 remaining profiler metrics (`FETCH_SIZE`,
`OCCUPANCY_PER_ACTIVE_CU`, `OCCUPANCY_PER_CU`, `OCCUPANCY_PERCENT`): the doc
already states "exported but always returns 0 on Radeon", which matches reality.
The `test_metric_coverage` failure for those is driven by the CI-side
`metrics-support.json` not listing W7900 — that file lives in the test repo, not
here, and is called out to the reporter separately.

### Alternatives considered

- Bundle with GPUOP-1019: rejected. GPUOP-1019's docs were already correct (all
  20 metrics `hypervisor_host: false` / `&cross;`); its fix was the gpuagent
  sentinel code, already committed on `gpuop-1019-sriov-suppress`. A doc change
  there would be a no-op.

## Scope

- **In scope:** metricslist.md + matrix corrections for W7900/Radeon on the 2
  genuinely-wrong metrics.
- **Out of scope:** CI-side `metrics-support.json` (test repo); the ROCM-27753
  STABLE_STD runtime behaviour (separate upstream ticket); any DME code change.

## Validation

- Doc-only change; no build. Cross-checked each edit against the CI evidence in
  the ticket (job 32412610) and against `metrics-support-matrix.yaml`.
- Prose for the 4 unchanged profiler metrics verified to already match observed
  W7900 behaviour.

## Risks and rollback

- Known risks: none (documentation only).
- Rollback plan: revert this commit.
