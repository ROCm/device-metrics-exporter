# ROCm 7.14.0 GA promotion: bump version refs to the public tarball

- **Date:** 2026-07-15
- **Author:** bhanu
- **Branch:** `chore/rocm-7.14-ga`
- **Related PR(s):** TBD
- **Related issue(s) / JIRA:** NO-JIRA (rc→GA promotion, amdsmi binaries identical)

## Context

TheRock 7.14 shipped as GA (https://github.com/ROCm/TheRock/releases/tag/therock-7.14).
The GA multi-arch tarball is now published on the public repo host
(`repo.amd.com/rocm/tarball-multi-arch/`). The exporter currently pins the release
candidate on the prerelease host
(`rocm.prereleases.amd.com/.../therock-dist-linux-multiarch-7.14.0rc2.tar.gz`,
landed in #1442 / GPUOP-970). This change promotes the pins to GA.

GA tarball:
`https://repo.amd.com/rocm/tarball-multi-arch/therock-dist-linux-multiarch-7.14.0.tar.gz`
(HTTP 200, 8,522,482,978 bytes / ~8.5 GB).

## Approach

Version-string + tarball-host change only. Edit `ROCM_VERSION` → `7.14.0` and
`ROCM_TARBALL_URL` → the GA tarball in both `Makefile` and `dev.env` (+ provenance
comment), and bump `assets/amd_smi_lib/version.txt` → `therock-7.14.0`. No asset
binary re-stage (see finding below). `make docker` (default `AMDSMI_FROM_TARBALL=0`)
consumes the existing staged assets; gpuagent compiles in-image at `GPUAGENT_COMMIT`.

## Scope

- **In scope:** `Makefile` + `dev.env` version/URL bump, `version.txt` string bump.
- **Out of scope:** amdsmi binary re-stage (GA ≡ rc2, no-op). `AMDSMI_BRANCH`
  (`release/therock-7.14`) / `AMDSMI_COMMIT` — already on the 7.14 release branch,
  source-build path only, unchanged. Profiler assets (`assets/rocprofiler*`,
  `assets/rocprofiler-sdk/config.yaml`) — built from rocprofiler source, not the
  therock tarball; untouched by a tarball promotion. `assets/version.yaml` /
  `assets/profiler_lib/version.txt` — stale metadata outside the build path. RVS
  tarball URL — already on repo.amd.com. gpuagent commit — no bump for a tarball
  promotion. DCM is a separate repo/change (same GA bump, tracked separately).

## Key finding: GA amdsmi binaries byte-identical to rc2

Stream-extracted the amdsmi payload from the GA `7.14.0` tarball and SHA256-compared
against the currently-staged rc2 assets (`assets/amd_smi_lib/x86_64/RHEL9/lib/`).
**All 7 tracked artifacts are byte-for-byte identical** — same identity relationship
that held rc1→rc2 (#1442). Only `version.txt` changes at the asset level.

| Asset | rc2 vs GA (SHA256) |
|---|---|
| `libamd_smi.so.26.5.0` | identical (`2441dcb9…f22a5ddd`) |
| `amdsmi.h` | identical (`d4a92854…9f5f8f4c`) |
| `librocm_sysdeps_drm.so.2` | identical (`cf203bcf…07877775`) |
| `librocm_sysdeps_drm_amdgpu.so.1` | identical (`5485dbf0…1825faf5`) |
| `librocm_sysdeps_mnl.so.0` | identical (`d73ed5e0…f373f2175`) |
| `librocm_sysdeps_nl_3.so.200` | identical (`f83b147a…88fdd10c1f9bac8`) |
| `librocm_sysdeps_nl_genl_3.so.200` | identical (`ab2dc13a…a64fc0c0`) |

Editing `version.txt` directly is equivalent to `make amdsmi-sync-assets` here and
avoids the 8.5 GB re-download.

## Validation

- GA tarball reachable (HTTP 200, 8.5 GB).
- amdsmi SHA256 identity vs staged rc2 (table above).
- `make docker` (standard, `AMDSMI_FROM_TARBALL=0`): build succeeds; gpuagent
  compiles in-image at `GPUAGENT_COMMIT`; amdsmi + 5 sysdeps staged; no
  `undefined symbol`.
- Mock smoke: `gpu_*` metrics served with non-zero values, no ErrZeroGPUs.
- **Real-HW (banff, MI300X):** deploy image, confirm live non-zero
  `gpu_average_package_power` / clock / VRAM, `/opt/rocm` → `/opt/rocm-7.14.0`,
  `amd-smi version` = 26.5.0, no ErrZeroGPUs / undefined-symbol in logs. Read
  `ServerPort` from the host's `/etc/metrics/config.json` (do not hardcode 5000).

## Risks and rollback

- Very low risk — only the version suffix + tarball host changed; amdsmi binaries
  unchanged from the already-validated rc2. Rollback: revert the Makefile/dev.env/
  version.txt bump.
