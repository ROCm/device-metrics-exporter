## Context

Adding release notes for the v1.5.1-beta release of the GPU Device Metrics Exporter.
This is a beta release — only the release notes are added; no image tags, Helm versions,
or other docs are updated.

## Approach

1. Prepend a `## v1.5.1-beta` section to `docs/releasenotes.md` above the v1.5.0 entry,
   following the existing format.

### Release note content decisions

- Version heading: `v1.5.1-beta`
- Platform support: two lines — existing platforms on ROCm 6.2+, new platforms on ROCm 7.13+
- Section order: features → known issues → platform support (matches existing format)
- Helm chart args: compact single bullet with two sub-flags as one-liners
- `--exit-on-rocpctl-error`: one-liner — exit instead of auto-disabling profiler on consecutive errors
- `--exit-on-agent-down`: one-liner with inline **Note:** for health monitoring continuity impact
- Known issues scoped to MI350P and Radeon AI for CPER and AFID; gfx_activity uptick is
  cosmetic with no workaround

### Alternatives considered

- Update image tags and Helm versions alongside release notes — rejected; this is a beta
  release and tag/version updates are out of scope to minimize churn.
- Promote Known Issues above features — rejected; beta label communicates risk adequately
  and deviating from the established format adds friction for readers familiar with it.

## Scope

- **In scope:**
  - `docs/releasenotes.md` — new v1.5.1-beta section
- **Out of scope:**
  - Image tag updates in any doc (docker.md, kubernetes-helm.md, etc.)
  - Helm chart version updates
  - `--privileged` flag additions
  - `docs/conf.py` version bump
  - Code changes

## Risks and rollback
